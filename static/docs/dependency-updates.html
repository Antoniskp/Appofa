Dependency Updates Guide

This document describes how to keep dependencies up to date, run security audits, and handle common upgrade tasks for this project.

Checking for Vulnerabilities

npm audit

To automatically fix vulnerabilities that don't require breaking changes:

npm audit fix

To fix all vulnerabilities, including those that require major version bumps (review output carefully before applying):

npm audit fix --force

Checking for Outdated Packages

npm outdated

This lists all packages where a newer version is available compared to what is installed (and what is specified in package.json).

Updating Direct Dependencies

To update a specific package to its latest compatible version (respecting semver ranges in package.json):

npm update <package-name>

To update all direct dependencies to their latest compatible versions:

npm update

To upgrade a specific package beyond the current semver range (e.g., to a new major version):

npm install <package-name>@latest

Package Overrides

The package.json contains an overrides section that forces specific versions of transitive dependencies to address known vulnerabilities:

"overrides": {
  "tar": "^7.5.8",
  "glob": "^13.0.0",
  "rimraf": "^6.0.1",
  "underscore": ">=1.13.8"
}

These should be reviewed periodically and removed once the parent packages have updated their own dependency ranges.

ESLint Configuration

This project uses ESLint v9 with the flat config format (eslint.config.js). The previous .eslintrc.js format is not used.

If you need to add new ESLint plugins or rules, edit eslint.config.js directly. The @eslint/js package (bundled with ESLint v9) provides js.configs.recommended, and the globals package provides language-specific global variables.

To run the linter:

npm run lint
